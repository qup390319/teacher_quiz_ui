知識節點路徑圖 — 兩個改造方向

水溶液單元 · 12 個節點 · 子主題 A（5 個，線性）+ 子主題 B（7 個，含 5-5/5-6 平行）

🌿 系統色變體 (6 種，貼合 spec-07)

使用 SciLens 主色票：米色紙底 / 木紋褐 / 主綠 / 暖橘 / 12 色節點分群（取自 theme.js 的 NODE_GROUP_COLORS）。整體與系統其他頁面協調，不再突兀。

F ★ 最新 深色發光技能樹 (Glowing Skill Tree)

仿《GreedFall》《God of War》《Apple Fitness+》風：深色底 + 發光節點 + 色彩分群。已啟用的節點會發光（總覽模式預設全亮）；未啟用則灰階暗化。子主題 A、B 用不同色系區分，終點節點金色更亮。

子主題 A（青藍光） 子主題 B（橘紅光） 終點（金光） 灰虛線 = 未啟用節點（例：教師未涵蓋）

深色 + 發光，最符合「遊戲技能樹」氛圍；總覽看會是滿屏光點
已啟用 / 未啟用可用「發光 vs 灰虛線」一眼區辨——非常適合「教師選定的節點」對比「未涵蓋節點」
未來易加入：節點點亮動畫、迷思題目數徽章、學生個別進度
深色背景與專案主視覺（白底 + 木框）反差較大，可能需要做成全螢幕「總覽彈窗」呈現

E ★ 推薦 技能樹風 (Skill Tree)

六角節點 + 階段欄位 + 箭頭連線。「階段 1 → 階段 6」沿橫軸排列，先後順序一目了然；節點極簡無裝飾；分岔 / 合流在階段 5–6 用對角線清楚表達。

子主題 A（線性） 子主題 B（含階段 5 平行分岔） ★ 黃框 = 該主題終點節點 ➜ 箭頭表示先後順序

階段欄位 + 箭頭，先後順序與層級關係一眼可辨
六角節點極簡、無裝飾，符合「不要插圖」需求
分岔 / 合流用對角線清楚畫出，比 Y 形管或繞湖更精確
未來易於擴充：每節點可以加「已完成」灰階、「鎖定」斜線、「進度」百分比，全套技能樹語彙都能套用

A 地鐵路線圖風 (Subway Map)

兩條彩色軌道，圓形站點清楚標出順序，平行階段（5-5 / 5-6）用真正的分岔再合流呈現。Lv.1–Lv.5 學習階段在底部，一眼看出進度。

子主題 A：溶解（線性） 子主題 B：酸鹼反應（含平行階段） 🏁 該主題終點 ● 分岔/合流交會點

節點順序、平行關係一眼可辨；視覺上「真的像圖」
預設只顯示短碼，hover 浮卡片顯示節點名稱（圖中 II-3-03 示意）
兩條軌道並排，需要的垂直空間比現在多一些

B 冒險地圖技能樹風 (Adventure Map)

符合 spec-07 日系手遊冒險風：蜿蜒虛線路徑、石碑/木牌節點、起點旗與終點寶箱。比 Mockup A 更有「闖關」氛圍，但結構較自由、平行段需要靠路徑彎度暗示。

🚩 起點　🎁 終點（寶箱） ● 岔路 / 合流標記 石碑顏色區分子主題

最符合專案的日系手遊冒險風 (spec-07)，「闖關」感很強，老師、學生都覺得親切
虛線小石路、彎曲路徑帶有手繪地圖味，差異化強
分岔處比 A 較不結構化，需靠「岔路」小標籤輔助說明
繪製成本比 A 高一些（路徑控制點要手調）

C 實驗室燒杯風 (Lab Bench)

完全貼合「水溶液」主題：節點 = 燒杯 / 錐形瓶 / 試管，連線 = 玻璃輸送管 + 流動氣泡。子主題 A 是清澈水溶液，子主題 B 用 pH 漸層色（藍→綠→黃→粉）呈現酸鹼變化。最具學科辨識度。

💧 燒杯 = 子主題 A 節點 🧪 錐形瓶 = 子主題 B 節點，色彩依 pH 漸層 玻璃管串接 + 氣泡 = 學習路徑 🏅 子主題終點

主題契合度滿分——一看就知道是「水溶液 / 酸鹼」單元
B 子主題用 pH 色階呈現酸→中→鹼，本身就是教學線索
學生會覺得「這是我的科學課本」，吸引力強
SVG 較複雜，繪製成本最高；視覺資訊密度高，第一眼可能略「滿」

D 寶可夢闖關路線圖 (Route Map)

復古 GBA 風 / RPG 地圖：草地、湖泊、樹林點綴。圓形「關卡石」用紅磚路串連，已通過會變星等。子主題 A、B 是兩條獨立路線。學生親近感最高、闖關感最強。

🏠 起點小屋　🏛 道館（終點） ⭐ 起始關　👑 BOSS 關 🟦 藍邊石 = 子主題 A　🟥 紅邊石 = 子主題 B 💧 湖泊（B 分岔自然繞過）

闖關感、遊戲感最強，學生會主動想「破關」
教師端用相同視覺，學生端可直接重用 → 完成節點變星等亮起
分岔用湖泊「擋住」自然要繞，比 A/B 兩條軌道更有故事性
整體偏 RPG，需確認指導教授可接受「童趣」程度
繪製工作量最大，但裝飾元素 (樹/石/湖) 一次畫好可重用

📌 建議

Mockup A (地鐵路線圖) — 推薦先做。結構清楚、實作快、平行階段最直觀。
Mockup B (冒險地圖) — 視覺最有特色、最貼專案調性，但分岔較難呈現。

可以考慮的折衷：「A 的結構 + B 的調性」——保留 A 的軌道與分岔邏輯，但把站點換成 B 的木牌石碑、把直線改成略帶弧度，這樣兩邊優點都有。
