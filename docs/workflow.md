# SciLens 系統工作流程（純業務邏輯）

> 本文件只描述「業務邏輯」與「使用者操作」，不涉及任何技術選型、框架、資料庫、API、檔案結構或實作細節。
> 文件末尾（§7）將所有「AI 介入節點」獨立編號為 `N1`、`N2` …，便於後續指定每個節點要使用的 AI / 檢索服務。

---

## 1. 角色

| 代號 | 角色 | 在系統中能做什麼 |
|------|------|------------------|
| T | 教師 | 管理班級與學生、出題、派發任務、檢視診斷結果 |
| S | 學生 | 接收任務、進行測驗、查看個人學習報告 |

---

## 2. 認證與帳號

### 2.1 帳號規則

- 每位教師擁有一個帳號（範例：`aaa001`）。
- 每位學生擁有一個帳號（範例：`115001`）。
- 帳號**預設密碼等同帳號本身**，學生首次登入後可自行修改。
- 教師在「學生管理」頁面可看到自己班級每位學生的帳號（並可重設密碼為預設值）。

### 2.2 登入流程

```
使用者開啟系統
    │
    ├─ 輸入「帳號」「密碼」
    │       │
    │       ├─ 驗證失敗 → 顯示錯誤訊息，停留於登入頁
    │       └─ 驗證成功 → 系統依帳號判定角色
    │
    ├─ 角色 = 教師 → 進入教師端首頁
    └─ 角色 = 學生 → 進入學生端首頁
```

### 2.3 登出

- 任一頁面皆可登出，登出後回到登入頁，並清除當前裝置的登入狀態。

## 3. 教師端工作流程

教師端共有四個主要任務區塊：班級管理、出題（迷思診斷）、派發任務、查看診斷結果。

### 3.1 班級與學生管理

```
教師進入「班級管理」
    │
    ├─ 瀏覽所有班級（每張班級卡片：名稱、年級、科目、學生人數）
    │
    └─ 點擊單一班級 → 進入班級詳情
        ├─ 檢視學生名冊（姓名、座號、帳號、初始密碼）
        ├─ 新增學生（自動配發帳號與初始密碼）
        ├─ 移除學生（軟性移除，保留歷史作答）
        └─ 重設某位學生密碼為預設值
```

### 3.2 出題：迷思診斷題組

「迷思診斷題組」由一組四選一題目組成；每個選項對應一條「正確答案」或一條「迷思概念」。

```
教師進入「題組庫」
    │
    ├─ 瀏覽既有題組（顯示標題、題數、狀態：草稿/已發布、建立日期）
    ├─ 可複製、刪除、重新命名既有題組
    │
    └─ 點擊「新增題組」 → 進入兩步驟出題精靈
        │
        ├─ 步驟一：選擇知識節點
        │   ├─ 系統依課綱列出所有可選節點（依層級分組）
        │   ├─ 教師勾選想涵蓋的節點
        │   └─ 進入下一步
        │
        └─ 步驟二：編輯題目
            ├─ 系統依「步驟一所選節點」載入候選題目
            ├─ 教師可編輯題幹、4 個選項文字、各選項對應的迷思
            ├─ 教師編輯選項文字時，系統可提供「文獻中學生真實說法」作為素材建議  ← AI 介入【見 N4】
            ├─ 教師可新增 / 刪除題目
            ├─ 儲存為「草稿」或直接「發布」
            └─ 完成後回到題組庫
```

### 3.3 派發任務

派發任務一律為**整班派發**：對象是整個班級的全部學生，教師指定截止日期後即派發給該班所有人。

```
教師進入「派發管理」
    │
    ├─ 以矩陣呈現：
    │   ├─ 列 = 題組
    │   ├─ 欄 = 班級
    │   └─ 每格顯示「完成率、已繳交/班級總人數、截止日期」或「未派發」
    │
    ├─ 點擊「未派發」格子 → 開啟整班派發視窗
    │   ├─ 選擇截止日期（必填）
    │   └─ 確認 → 對該班所有學生建立派題記錄
    │
    └─ 點擊「已派發」格子 → 修改截止日期 / 取消派題
```

### 3.4 查看診斷結果

```
教師進入「診斷結果」
    │
    ├─ 頂部選擇要檢視的「題組」（切子分頁時保留此選擇）
    │
    ├─ 子分頁 A：所有班級總覽
    │   ├─ AI 所有班級診斷摘要（年級健康狀態、跨班觀察、優先介入順序、教學行動建議）  ← AI 介入【見 N1】
    │   ├─ 4 個整體指標卡：涵蓋班級、平均完成率、平均掌握率、需關注班級
    │   └─ 班級分布散佈圖（完成率 × 掌握率）
    │
    ├─ 子分頁 B：各班學習狀況
    │   ├─ 每班三項核心指標卡片（完成率、掌握率、高頻迷思數）
    │   └─ 點擊班級卡 → 切到「各班詳細報告」並帶入該班
    │
    ├─ 子分頁 C：知識節點跨班比較
    │   └─ 同一概念節點各班通過率並排呈現（含 70% 掌握門檻線）
    │
    ├─ 子分頁 D：跨班高頻迷思
    │   ├─ 高頻迷思排行（依所有班級平均持有率排序）
    │   └─ 班級 × 迷思 熱力圖
    │
    └─ 子分頁 E：各班詳細報告
        ├─ 上方：班級派題完成率清單（被選中的班級高亮）
        └─ 下方：該班完整報告
            ├─ 4 個指標卡（參與學生、完成率、平均掌握率、最高風險迷思）
            ├─ 班級 AI 診斷摘要 + 本週行動清單  ← AI 介入【見 N2】
            ├─ 各概念掌握程度長條圖
            ├─ 迷思概念分佈（可展開該迷思對應的學生名單）
            └─ 題目明細矩陣（每題各選項選答分佈與對應迷思）
```

## 4. 學生端工作流程

學生端有兩個入口：診斷測驗、學習報告。

### 4.1 學生首頁

```
學生登入後進入學生首頁
    │
    ├─ 「📝 診斷測驗」區塊：列出教師派發的診斷任務（含截止日、是否已完成）
    └─ 「📊 學習報告」區塊：列出歷次完成的測驗結果
```

### 4.2 進行診斷測驗（對話式）

完整作答分為四個階段：「開場 → 選擇題作答 → AI 開放式追問 → 完成」

```
學生點選一份「診斷測驗」任務
    │
    ├─ 階段 A｜開場
    │   └─ 系統逐則顯示 3 則歡迎訊息，告知接下來會分兩個環節
    │
    ├─ 階段 B｜選擇題作答（不顯示對錯）
    │   ├─ 系統逐題顯示「節點名稱 + 題號 + 題幹」
    │   ├─ 學生從 4 個選項擇一作答
    │   ├─ 系統記錄該選項對應的「正確 / 迷思代號」，但不告訴學生對錯
    │   └─ 全部題目作答完 → 進入下一階段
    │
    ├─ 階段 C｜AI 開放式追問（每題 1～3 輪）
    │   │
    │   ├─ 過渡訊息：「選擇題的部分結束了，接下來我想跟你聊聊⋯」
    │   │
    │   ├─ 對「每一道題」都進行追問（不論答對答錯）：
    │   │   │
    │   │   ├─ 第 1 輪：AI 用開放式問句問學生「為什麼選這個答案」  ← AI 介入【見 N3】
    │   │   │   └─ 學生用自由文字回覆
    │   │   │
    │   │   ├─ 第 2 輪：AI 依學生第 1 輪回答的內容動態決定追問方向  ← AI 介入【見 N3】
    │   │   │   ├─ 情境 A（回答模糊）：給具體情境引導學生再想
    │   │   │   ├─ 情境 B（回答呼應某個迷思）：用對比情境挑戰
    │   │   │   ├─ 情境 C（答對且理由合理）：延伸情境確認穩固
    │   │   │   └─ 情境 D（答對但理由模糊）：遷移到不同情境再問
    │   │   │
    │   │   └─ 第 3 輪（必要時）：再追問迷思來源、二選一比較或遷移測試  ← AI 介入【見 N3】
    │   │
    │   ├─ 每題結束後，AI 對該題產出最終診斷：
    │   │   ├─ 「最終狀態」：正確 / 持有迷思 / 不確定
    │   │   ├─ 「推理品質」：穩固 / 部分理解 / 薄弱 / 用猜的
    │   │   └─ 「狀態變化」：確認 / 升級（從錯升正確）/ 降級（從對降為猜）/ 新發現
    │   │
    │   └─ 系統依「狀態變化」回頭修正階段 B 的紀錄：
    │       ├─ 升級 → 該題改為正確
    │       └─ 降級 → 該題改為對應的迷思代號
    │
    └─ 階段 D｜完成
        ├─ 顯示完成訊息
        └─ 自動帶到「學習報告」
```

### 4.3 學習報告（學生端）

```
學生在學習報告中可看到
    │
    ├─ 掌握 / 待更新 概念數
    │
    ├─ 被確認的迷思概念清單（每條包含）
    │   ├─ 迷思標籤（簡短）
    │   ├─ 學生端說明（用學生看得懂的話描述為什麼這想法不對）
    │   ├─ 「你在對話中提到」引用：摘錄學生在追問時的關鍵回覆
    │   └─ 學習提示：建議怎麼修正
    │
    ├─ 「答對了，但可以更深入」黃色區塊
    │   └─ 列出推理品質為「薄弱」或「用猜的」的題目
    │
    └─ 下一步學習建議（系統依當前迷思組合推薦複習方向）
```

## 5. 跨角色的端對端資料流

```
教師建立診斷題組（草稿 → 發布）
        │
教師派發診斷題組給班級
        │
學生收到任務 → 完成診斷測驗（含 AI 追問）
        │
系統儲存：每題作答、每輪追問對話、每題最終診斷
        │
教師查看診斷結果（總覽 / 班級 / 節點 / 高頻迷思 / 班詳）
        │
（資料回到教師端：協助下一輪派發決策；資料回到學生端：累積個人學習報告）
```

## 6. 系統需要持續累積的資料（業務視角，不含技術細節）

| 資料類別 | 內容 | 由誰產生 | 給誰看 |
|----------|------|----------|--------|
| 班級與學生名冊 | 班級、學生姓名、座號、帳號 | 教師 | 教師（管理）、學生（自身資料） |
| 診斷題組 | 題目、選項、迷思對應 | 教師 | 教師（編輯）、學生（作答） |
| 派發紀錄 | 哪份題組派給哪班、截止日、完成率 | 教師 | 教師、學生（看到任務） |
| 學生作答 | 每題選項、追問對話、最終診斷 | 學生 | 學生（自己的報告）、教師（班級報告） |
| 學生密碼變更紀錄 | 密碼變更時間、是否已被重設為預設 | 學生 / 教師 | 教師（管理） |

## 7. AI 介入節點清單（編號）

下表列出系統中所有需要 AI 介入的節點。每個節點都有獨立編號（`Nx`），便於後續逐一指定每一個節點要使用的 AI 服務或檢索來源。

### N1｜所有班級診斷摘要（教師端）
- **觸發時機**：教師進入「診斷結果 → 所有班級總覽」子分頁時。
- **輸入素材**：當前選定題組的所有班級作答彙整、各班指標、跨班節點通過率、跨班高頻迷思排行。
- **AI 任務**：
  1. 描述「年級整體健康狀態」一段話。
  2. 指出「跨班共同的觀察」與「特別需要關注的班級」。
  3. 提出「優先介入順序」（先處理哪些迷思 / 哪些班）。
  4. 給出「具體行動建議」（例如：可採行的教學示範、追問題、補充教材）。
- **輸出**：給教師看的中文段落式摘要 + 條列行動建議。

### N2｜單班診斷摘要（教師端）
- **觸發時機**：教師進入「診斷結果 → 各班詳細報告」並選定一個班級。
- **輸入素材**：該班全部學生作答彙整、各概念掌握程度、迷思分佈、題目明細矩陣。
- **AI 任務**：
  1. 描述該班整體學習狀態與最突出的問題。
  2. 列出「本週行動清單」（建議下一步派題 / 補充教學）。
  3. 點名需要特別關注的學生群（依迷思組合）。
- **輸出**：班級級別的中文摘要 + 行動清單。

### N3｜診斷測驗的開放式追問（學生端，第二階段）
- **觸發時機**：學生完成所有選擇題後，系統對每一題進行 1~3 輪開放式追問。
- **輸入素材**：題目、學生所選的選項、該選項對應的迷思代號、學生本輪自由文字回覆、該題到目前為止的完整對話。
- **AI 任務**：
  1. 第 1 輪：以開放式問句探詢學生選擇的理由。
  2. 第 2 輪：依學生回答動態判讀屬於哪一種情境（模糊 / 呼應迷思 / 答對且穩固 / 答對但模糊），並產出對應追問。
  3. 第 3 輪：必要時追問迷思來源、提供二選一比較或遷移情境。
  4. 終止後產出：最終狀態、推理品質、狀態變化（並驅動系統回頭修正第一階段的紀錄）。
- **輸出**：每一輪都產生 1 則對話氣泡 + 最終診斷結構化結果。

### N4｜出題輔助（教師端）
- **觸發時機**：教師在診斷題組出題精靈步驟二編輯選項文字時。
- **輸入素材**：當前題目對應的知識節點 ID、當前選項要對應的迷思 ID、教師目前已輸入的選項文字。
- **AI 任務**：
  1. 從外部知識庫檢索「該迷思在文獻中的真實描述語句」與「研究中學生實際說法的範例」。
  2. 建議 1~3 條可作為「干擾選項」的選項文字候選，貼合該迷思的研究描述。
  3. 同時回傳對應的文獻引用，供教師決定是否採用。
- **輸出**：候選選項文字（教師可一鍵採用、改寫或忽略）+ 引用清單。

## 8. 對應對照表（給後續設計參考）

| 節點 | 主要使用方 | 是否要求即時回應 | 是否需檢索外部知識 | 指派的 AI 服務 |
|------|------------|-------------------|---------------------|----------------|
| N1 所有班級摘要 | 教師 | 否（可批次 / 可快取） | 是（檢索文獻知識庫） | **RAGFlow Agent** |
| N2 單班摘要 | 教師 | 否（可批次 / 可快取） | 是（檢索文獻知識庫） | **RAGFlow Agent** |
| N3 診斷追問 | 學生 | 是（對話即時） | 否（純對話推理） | 純 LLM（無 RAG） |
| N4 出題輔助 | 教師 | 否（教師備課時） | 是（檢索文獻知識庫） | **RAGFlow Agent** |

> **服務分工原則**：
> - **RAGFlow Agent**（N1 / N2 / N4）：需要從外部研究文獻檢索內容，附引用來源
> - **純 LLM**（N3）：純對話推理，不需檢索外部知識，由系統自行組裝 prompt 與對話歷史
>
> RAGFlow 知識庫內容為「水溶液迷思概念研究文獻整理」三份文件（一般整理 / 國小學童研究 / 後設研究）。
