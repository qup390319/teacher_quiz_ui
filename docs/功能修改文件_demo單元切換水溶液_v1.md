## 1. 修改背景與目標

### 1.1 背景
本系統目前的 demo 單元為國中三年級自然科學「**溫度與熱**」，含 8 個知識節點與 2 份示範考卷。隨著研究方向調整，需要將 demo 單元全面更換為國小自然科學「**水溶液**」單元，以對應新的研究對象（國小高年級學童）。

### 1.2 目標
- 將系統內所有與「溫度與熱」相關的示範內容（單元名稱、知識節點、迷思概念、考卷題目、班級年級）替換為「水溶液」單元。
- 維持目前所有功能行為與頁面流程**完全不變**，僅替換 demo 內容。
- 教師與學生在頁面上看到的單元名稱、節點清單、題目、迷思說明，全面更新為水溶液主題。

### 1.3 不變更的部分
- 系統的所有功能（出題精靈、派題、診斷報告、學生作答、迷思確認、學習報告等）行為與流程**完全保留**。
- 頁面排版、互動方式、圖表種類、色彩主題不動。
- 班級數量（3 班）、各班學生人數（20/18/22 人）不變。
- 兩份示範考卷的數量與題數設計不變（兩份各自獨立的迷思診斷考卷，恰好涵蓋同一組知識節點）。

## 5. 迷思概念設計

**每個知識節點皆對應 4 個學童常見迷思**，12 個節點合計 **48 條迷思**。內容主要取材自《國小學童對水溶液的迷思概念類型與成因之研究》、《國內學生自然科學迷思概念研究之後設研究》與「自我解釋-討論-再解釋」（SDR）學習策略相關研究。

> **編號規則**：迷思編號 `M{XX}-{Y}` 中，`M01–M05` 對應子主題 A（溶解）的 5 個節點，`M06–M12` 對應子主題 B（酸鹼）的 7 個節點，與節點清單順序一致；`Y` 為該節點下的迷思序號（1–4）。

---

### 5.1 子主題 A：水溶液中的變化（溶解）

#### 節點 1：INe-II-3-01 生活中溶解的現象
| 編號 | 迷思內容 |
|---|---|
| M01-1 | 認為只要把固體加進水裡就算「溶解」，不論最後是否仍能看到顆粒或沉澱 |
| M01-2 | 把「混合」當成「溶解」（例如沙子加入水中也叫溶解） |
| M01-3 | 認為一定要攪拌或加熱才會發生溶解，否則物質不會溶 |
| M01-4 | 認為只有「看不見」的才算溶解，看得到顆粒的就不算 |

#### 節點 2：INe-II-3-02 溶解是物質在水中變小而看不見，會出現沉澱
| 編號 | 迷思內容 |
|---|---|
| M02-1 | 溶解後物質會消失不見 |
| M02-2 | 把「溶解」與「融化」當成同一件事 |
| M02-3 | 溶解後物質會蒸發到空氣中 |
| M02-4 | 糖溶於水是化學變化，溶解後就不再是原來的糖 |

#### 節點 3：INe-II-3-03 攪拌可以增加溶解速度但不能增加溶解量
| 編號 | 迷思內容 |
|---|---|
| M03-1 | 攪拌可以增加水中能溶解的量 |
| M03-2 | 沒有攪拌，物質就不會溶解 |
| M03-3 | 攪拌可以讓已沉澱在杯底的物質重新溶解 |
| M03-4 | 攪拌與溶解速度沒有關係 |

#### 節點 4：INe-II-3-04 不同物質在水中的溶解程度不同
| 編號 | 迷思內容 |
|---|---|
| M04-1 | 認為所有物質在水中的溶解能力都一樣 |
| M04-2 | 認為固體才能溶解，液體不能溶於水 |
| M04-3 | 顆粒小的物質一定能溶解的量也比較多（混淆「溶解速率」與「溶解量」） |
| M04-4 | 認為「比較貴」或「比較乾淨」的物質溶解能力比較強 |

#### 節點 5：INe-II-3-05 同一物質有固定溶解量，超過會出現沉澱
| 編號 | 迷思內容 |
|---|---|
| M05-1 | 加熱讓鹽暫時溶解，過一段時間又會全部沉澱回來 |
| M05-2 | 沉澱只是因為溶質比較重沉到底部，與溶解量上限無關 |
| M05-3 | 只要持續加水或加熱，水可以無限溶解任何量的糖 |
| M05-4 | 溶解的物質呈破碎狀比較輕，會浮到水面 |

### 5.2 子主題 B：酸鹼反應

#### 節點 6：INe-Ⅲ-5-1 水溶液包含溶質和溶劑
| 編號 | 迷思內容 |
|---|---|
| M06-1 | 認為水溶液裡只有「水」一種東西，看不到溶質就代表它不存在 |
| M06-2 | 把「溶質」與「溶劑」混淆，分不清哪一個比較多 |
| M06-3 | 認為水以外的液體不能當作溶劑 |
| M06-4 | 認為溶質一定要是固體（液體或氣體不能當溶質） |

#### 節點 7：INe-Ⅲ-5-2 辨別生活中的水溶液
| 編號 | 迷思內容 |
|---|---|
| M07-1 | 認為所有透明的液體都是水溶液（包含純水） |
| M07-2 | 認為不透明的液體（如牛奶、果汁）一定不是水溶液 |
| M07-3 | 認為杯底有沉澱的混合物也算水溶液 |
| M07-4 | 認為只有市售飲料才算水溶液，自製的不算 |

#### 節點 8：INe-Ⅲ-5-3 石蕊試紙的正確使用方法
| 編號 | 迷思內容 |
|---|---|
| M08-1 | 認為石蕊試紙要泡很久才能看出顏色變化 |
| M08-2 | 認為要用力搓揉試紙才會變色 |
| M08-3 | 認為一張試紙可以重複測量多種不同溶液 |
| M08-4 | 認為試紙顏色越深就代表酸性或鹼性越強 |

#### 節點 9：INe-Ⅲ-5-4 用石蕊試紙檢驗水溶液的酸鹼性
| 編號 | 迷思內容 |
|---|---|
| M09-1 | 食鹽水是鹹的，所以是鹼性（用味覺判斷） |
| M09-2 | 沒有刺鼻味的水溶液一定是中性 |
| M09-3 | 名稱裡有「酸」字才是酸性，否則就一定不是酸性 |
| M09-4 | 用嘴巴嚐就能判斷水溶液的酸鹼性 |

#### 節點 10：INe-Ⅲ-5-5 自製酸鹼指示劑（含花青素植物、紫色高麗菜）
| 編號 | 迷思內容 |
|---|---|
| M10-1 | 認為只有市售試紙才能驗酸鹼，植物汁液不能拿來做指示劑 |
| M10-2 | 認為紫色高麗菜汁本身就是酸性或鹼性 |
| M10-3 | 認為植物汁變色只是因為被稀釋，與酸鹼無關 |
| M10-4 | 認為所有紫色植物的汁液都會用完全一樣的方式變色 |

#### 節點 11：INe-Ⅲ-5-6 自製酸鹼指示劑（蝶豆花）
| 編號 | 迷思內容 |
|---|---|
| M11-1 | 認為蝶豆花茶加檸檬汁變紫紅色是「染色」現象，與酸鹼無關 |
| M11-2 | 認為蝶豆花茶顏色改變代表變質、不能再飲用 |
| M11-3 | 認為只有蝶豆花有這種變色能力，其他植物都不行 |
| M11-4 | 認為加越多檸檬汁，蝶豆花茶顏色只是變淡，與酸度多寡無關 |

#### 節點 12：INe-Ⅲ-5-7 酸鹼可以用來解決生活中的問題
| 編號 | 迷思內容 |
|---|---|
| M12-1 | 酸與鹼都很危險，所有酸鹼物質都不能接近 |
| M12-2 | 水溶液濃度越高就一定越強酸或越強鹼 |
| M12-3 | 酸與鹼中和後，兩種物質就完全消失了 |
| M12-4 | 名稱裡有「酸」字的物質就一定有腐蝕性、有毒 |

## 6. 兩份考卷的完整題目

### 6.1 考卷一：水溶液 · 迷思診斷（第一次）

#### 第 1 題（節點 INe-II-3-02 溶解現象）
**題幹**：媽媽把一匙糖加入熱開水並攪拌，糖看不見了。這時候糖到哪裡去了？

| 選項 | 內容 | 對應診斷 |
|---|---|---|
| A | 糖變成肉眼看不見的小粒子分散在水裡，仍然存在於水中 | ✅ 正確 |
| B | 糖完全消失了，已經不在水裡 | M02-1 |
| C | 糖像冰塊一樣融化了，已經變成水的一部分 | M02-2 |
| D | 糖蒸發到空氣中了，所以看不見 | M02-3 |

#### 第 2 題（節點 INe-II-3-03 攪拌與溶解）
**題幹**：小明在 100 毫升的水中加入很多糖，攪拌一陣子後杯底仍有糖沒溶完。下列敘述何者最正確？

| 選項 | 內容 | 對應診斷 |
|---|---|---|
| A | 只要攪拌得更用力、攪久一點，剩下的糖就一定能全部溶解 | M03-1 |
| B | 攪拌只能加快糖溶解的速度，無法讓更多的糖溶進這杯水裡 | ✅ 正確 |
| C | 沒有攪拌的話，糖根本不會溶解 | M03-2 |
| D | 持續攪拌，沉在杯底的糖也會跟著重新溶解進水裡 | M03-3 |

#### 第 3 題（節點 INe-II-3-05 溶解量上限）
**題幹**：小華把鹽一直加進一杯水中，加到最後杯底開始有鹽溶不掉。下列哪個說法最正確？

| 選項 | 內容 | 對應診斷 |
|---|---|---|
| A | 一杯水能溶解的鹽量有上限，超過就會沉澱 | ✅ 正確 |
| B | 鹽沉到杯底是因為鹽比水重，與溶解量上限無關 | M05-2 |
| C | 只要不斷加熱與加水，這杯水可以溶解任何量的鹽 | M05-3 |
| D | 鹽是破碎狀比較輕會浮上來，沒看到就是已經溶完了 | M05-4 |

#### 第 4 題（節點 INe-Ⅲ-5-4 石蕊試紙檢驗酸鹼）
**題幹**：要判斷一杯透明的水溶液是酸性、中性還是鹼性，最適合的方法是？

| 選項 | 內容 | 對應診斷 |
|---|---|---|
| A | 用嘴巴嚐看看，鹹的就是鹼性 | M09-1 |
| B | 聞看看，沒有刺鼻味就是中性 | M09-2 |
| C | 用石蕊試紙浸入水溶液中，依顏色變化判斷 | ✅ 正確 |
| D | 看名稱中有沒有「酸」字，沒有就一定不是酸性 | M09-3 |

#### 第 5 題（節點 INe-Ⅲ-5-7 酸鹼於生活）
**題幹**：水管被堵住時，可以倒入鹼性的通樂（含氫氧化鈉）來通水管。下列敘述何者最正確？

| 選項 | 內容 | 對應診斷 |
|---|---|---|
| A | 鹼性物質都有毒，絕對不能拿來做家庭清潔 | M12-1 |
| B | 通樂能把水管裡的油脂等阻塞物分解掉 | ✅ 正確 |
| C | 通樂越濃就一定越強鹼，越稀就是弱鹼 | M12-2 |
| D | 鹼性遇到酸性會把彼此完全消滅變成虛無 | M12-3 |

### 6.2 考卷二：水溶液 · 迷思診斷（第二次）

#### 第 1 題（節點 INe-II-3-02 溶解現象）
**題幹**：把鹽放進水裡攪一攪後鹽看不見了，過一會兒水嚐起來仍是鹹的。這代表？

| 選項 | 內容 | 對應診斷 |
|---|---|---|
| A | 鹽已經不是原來的鹽，溶解過程把鹽變成另一種物質 | M02-4 |
| B | 鹽變成肉眼看不見的小粒子均勻分散在水中 | ✅ 正確 |
| C | 鹽真的不見了，只是水的味道剛好變鹹 | M02-1 |
| D | 鹽蒸發了，鹹味是水自己變的 | M02-3 |

#### 第 2 題（節點 INe-II-3-03 攪拌與溶解）
**題幹**：兩杯一樣的水各加入 5 公克糖，A 杯不攪拌、B 杯快速攪拌。最後比較兩杯，會發現？

| 選項 | 內容 | 對應診斷 |
|---|---|---|
| A | 兩杯能溶解的糖量相同，只是 B 杯溶解得比較快 | ✅ 正確 |
| B | B 杯能溶解的糖比 A 杯多，因為攪拌會讓水容納更多糖 | M03-1 |
| C | A 杯的糖完全沒有溶解，因為沒攪拌就不會溶解 | M03-2 |
| D | 兩杯溶解情況完全一樣，攪拌不會加快也不會增加 | M03-4 |

#### 第 3 題（節點 INe-II-3-05 溶解量上限）
**題幹**：在室溫下，把糖一直加入同一杯水並攪拌。當水中已經溶不下時你會看到？

| 選項 | 內容 | 對應診斷 |
|---|---|---|
| A | 多餘的糖會沉到杯底，因為水能溶解的糖量有極限 | ✅ 正確 |
| B | 多餘的糖只是因為比較重而沉底，再攪拌、加熱還會繼續溶 | M05-2 |
| C | 水可以無限溶解糖，看到沉澱代表還沒攪拌均勻 | M05-3 |
| D | 加熱可以讓糖暫時全溶，但放一段時間後又會全部沉澱出來 | M05-1 |

#### 第 4 題（節點 INe-Ⅲ-5-4 石蕊試紙檢驗酸鹼）
**題幹**：將紫色石蕊試紙浸入小蘇打水中，試紙變成藍色。這代表小蘇打水是？

| 選項 | 內容 | 對應診斷 |
|---|---|---|
| A | 酸性，因為「小蘇打」這個名稱沒有「酸」字 | M09-3 |
| B | 鹼性，因為紫色石蕊試紙在鹼性溶液中會變成藍色 | ✅ 正確 |
| C | 中性，因為小蘇打水沒有刺鼻味 | M09-2 |
| D | 鹼性，因為小蘇打水嚐起來不是酸的也不是鹹的 | M09-4 |

#### 第 5 題（節點 INe-Ⅲ-5-7 酸鹼於生活）
**題幹**：被蜜蜂叮咬時皮膚會痛癢，這是因為蜜蜂的毒液偏酸性。下列建議何者最合適？

| 選項 | 內容 | 對應診斷 |
|---|---|---|
| A | 不要碰它，所有酸都很危險、有毒，碰了會更糟 | M12-1 |
| B | 塗一點偏鹼性的小蘇打水，可以幫助緩解 | ✅ 正確 |
| C | 馬上塗檸檬汁，因為酸與酸可以互相消滅 | M12-3 |
| D | 因為有「酸」字，皮膚一定會被腐蝕壞死，要立刻送醫 | M12-4 |

## 13. 風險與後續注意事項

1. **學生端迷思確認的文字情境化**：水溶液單元的迷思（如「溶解＝融化」）對國小學生需用更具體的例子說明（如「你是不是覺得糖在水裡像冰塊一樣融化了？」），實作時需逐條斟酌語氣。
2. **跨班作答分佈的真實性**：目前各班作答分佈為手工設計的模擬資料，僅用於畫面 demo，不代表真實學生作答狀態。此設計沿用，需在報告頁的「目前狀況說明」中保留警示文字。
3. **未出現在示範考卷的 7 個節點**：12 個節點中有 7 個未被示範考卷直接使用，但這些節點仍**完整提供 4 個學生常見迷思**（見 §5），會顯示於「知識地圖」、「節點詳細頁」、「教師端教學策略卡片」、「出題精靈節點清單」中，供教師作為自編題目與教學設計的素材。
4. **舊單元資料是否保留**：本次修改為**完全替換**，不保留「溫度與熱」單元的 demo 資料。若日後需要回切，需從版本控制歷史還原。
